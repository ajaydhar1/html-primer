What is HTML?

HTML stands for Hypertext Markup Language. It's the standard markup language used to create webpages.

Don't get confused by the word "markup". A markup language is just one that uses tags to provide information about it's contents. In the case of HTML, the tags are used to tell the browser how to show the contents of the webpage.

The latest version of HTML is HTML5.

HTML Tags

An HTML tag is a fundamental component of HTML. Each tag is like a word in the HTML language. There are approximately 100 different tags in HTML5, but only 30 or 40 are used regularly. We'll cover the 38 most commonly used tags in this document.

Note, the terms "tag" and "element" are often used interchangably to mean the same thing, which we describe next. However, in this document, we tried to use the term "tag" instead of "element" wherever possible, just because that's our preference. The term just seems a little more clear to us.

HTML tags consist of angle brackets ('<' and '>'), which enclose the name of the tag. And the tags come in pairs: an opening tag and a closing tag. The opening tag specifies the beginning of an element, and the closing tag denotes the end of that element. The content of the element is placed between the opening and closing tags.

For example:

<p>This is a paragraph.</p>

The opening tag is:

<p>

And the closing tag is:

</p>

Note that the name of the closing tag starts with a "/", like </p>. That's just to show that it is a closing tag.

That is the very basics of how an HTML tag works.

Now that we've got that out of the way, let's dive into the 38 HTML tags you'll need to know to kickstart your learning!

Syntax:

Note: In the context of computer programming, the term "syntax" refers to the proper way that the lines of code should be written, according the language you are using. It's like the grammer that must be followed to use the language.

For example, to create a variable and set it equal to 5 in Python, you would write x = 5, but in PHP you would write $x = 5;. 3. Inline Tags:

Syntax:

  1. <strong>bold word</strong>
  2. <b>the old way of bolding</b>

Description:

Inline tags are typically used within larger blocks of text to apply specific, visual formatting to the parts of the text that the tags surround.

  1. <strong>: Surrounds text that should be displayed in a bold font.
  2. <b>: Surrounds text that should be displayed in a bold font.
  3. <i>: Surrounds text that should be displayed in an italic font style.
  4. <em>: Surrounds text that should be emphasized, often displayed in italics by default.
  5. <mark>: Surrounds text that should be highlighted for reference or emphasis, often displayed with a background color.
  6. <sub>: Makes subscript text, typically rendered slightly below the normal baseline.
  7. <sup>: Makes superscript text, typically rendered slightly above the normal baseline.
  8. <u>: Surrounds text that should underlined.
  9. <span>: Used as a generic container for inline tags for applying styles to the text.

4. Anchors (aka Links):

Syntax:

<a href="https://www.example.com">Example link</a>

Description:

The anchor tag is used to create links, allowing users to navigate between different web pages or sections within the same page. When a user clicks on a link, it directs their browser to the URL specified in the href attribute of the <a> tag.

Here's a breakdown of the anchor tag and its two main attributes.

Attributes are key/value pairs that are included within the opening tag of an HTML element and adjusts the behavior or display of the HTML element. They consist of a name followed by an equal sign (=) and a value enclosed in double or single quotation marks.

Back to the anchor tag:

  • <a>: This is the opening tag of the anchor element.
  • href: Stands for "hypertext reference". This attribute specifies the destination URL, or the location to which the link should navigate. It can be an absolute URL (e.g., "https://www.example.com") or a relative URL (e.g., "page.html" or "#section"). Absolute URLs contain the full web address, including the protocol (e.g., "https://") and the domain name (e.g., "www.example.com"). Relative URLs are used for linking to a webpage or resource (resources include things like CSS file, JS file, image, etc.) in the same domain, and specify the location of the file relative to the location of the current file in the website (or project folder).
  • target: Specifies where to open the linked document. If the attribute is not included, the link is opened in the same browser tab by default. To open the link in a new tab, set the attribute to target="_blank">.

5. Ordered Lists:

Syntax:

<ol>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ol>

Description:

The <ol> tag stands for "ordered list". It's used to create a list of items, where each item is automatically numbered in sequence. Ordered lists are typically used when the order of the items is important.

<ol> is the outer tag. In between the opening and closing <ol> tags, there are <li> tags. <li> stands for "list item", and is used to define each item in the list.

6. Unordered Lists:

Syntax:

<ul>
    <li>Bullet item 1</li>
    <li>Bullet item 2</li>
    <li>Bullet item 3</li>
</ul>

Description:

The <ul> tag stands for "unordered list". It's used to create a list of items, where each item is prepended with a bullet point. Unordered lists are typically used when the order of the items is not important.

<ul> is the outer tag. In between the opening and closing <ul> tags, there are <li> tags. <li> stands for "list item", and is used to define each item in the list.

Note: The syntax for the <ol> and <ul> tags is very similar. The only difference is their outer tag: <ol> for ordered lists and <ul> for unordered lists. Both use <li> to list the items.

Syntax:

<video width="640" height="480" src="https://archive.org/download/Popeye_forPresident/Popeye_forPresident_512kb.mp4" controls>
    Sorry, your browser doesn't support HTML5 video.
</video>

Description:

The video tag is used to embed video content directly into web pages without requiring third-party plugins.

The two main attributes are:

  • src: Specifies the URL of the video file. It can be in various formats like MP4, WebM, or Ogg.
  • controls: This attribute adds standard playback controls to the video player, allowing users to play, pause, seek, and adjust the volume of the video.

Fallback Content: The text or elements placed between the opening and closing <video> tags are displayed in browsers that do not support the <video> tag.

8. Tables:

Syntax:

<table>
    <thead>
        <tr>
            <th>Numbers</th>
            <th>Letters</th>
            <th>Colors</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>1</td>
            <td>A</td>
            <td>Red</td>
        </tr>
        <tr>
            <td>2</td>
            <td>B</td>
            <td>Green</td>
        </tr>
        <tr>
            <td>3</td>
            <td>C</td>
            <td>Blue</td>
        </tr>
    </tbody>
</table>

Description:

The HTML <table> tag is used to create tables, which are a way to display data in rows and columns. Tables are commonly used to present information in a structured format, such as schedules, pricing lists, and data comparisons. The format looks similar to Microsoft Excel.

Basic Structure: The <table> tag defines the beginning of the table, and the </table> tag denotes the end of the table. Inside the table, you define rows using the <tr> tag (table row), and within each row, you define cells using the <td> tag (table data).

Headers: To define header cells in a table, you use the <th> tag (table header) instead of <td>. Header cells are typically displayed in bold and centered by default.

Note, optionally, you can place the header tags inside a <thead> tag and the body of the table inside a <tbody> tag to make the table more readable to other developers, and it also makes it easier to style.

Syntax:

The <img> tag in HTML is used to embed images into web pages. It stands for "image" and is a self-closing tag, meaning it does not require a separate closing tag.

12. Forms:

Syntax:

<form action="process.php" method="get">
     <h4>Personal Information</h4>
     <input name="name" id="name"/>
     <input name="dob" id="dob" type="date"/>
     <button type="submit">Submit</button>
 </form>

Description:

In HTML, the <form> element is a slightly advanced element that allows users to input data and send it to the server for processing. This information can include things like their name, email, birthdate, etc. When the server receives this data, it can save it in a database, use it to send back a personalized message, or many other things.

Forms are a fundamental part of web development, enabling various types of user interactions such as logging in, registering and submitting feedback.

At its core though, forms are just a tag that surround input elements, including a submit button. The submit button has an internal mechanism that when clicked, sends the data in the form fields to the webpage that is specified in the action attribute in the <form> tag. The webpage is usually a server-side page that is written in a server-side language like PHP.

In the example in the Syntax section above, the form fields will be sent to the server-side page "process.php", which can then do something with the user data, such as saving it to a database.

Here's how the <form> element works:

1. Basic Structure: The <form> tag is a container for form elements such as text inputs, checkboxes, radio buttons, and buttons. It has an opening <form> tag and a closing </form> tag. Inside the form, you place the elements that you want to include in the form to collect information from the user.

2. Attributes: The <form> element supports several attributes that control its behavior. The two main attributes are:

  • action: Specifies the URL where the form data should be submitted.
  • method: Specifies the HTTP method to be used when submitting the form data. Common values are "GET" and "POST". This a slightly advanced concept that we can get to later.

3. Form Elements: Inside the <form> tag, you can include various form elements such as <input>, <textarea>, <select>, and <button>. These elements allow users to input data into the form.

4. Submission: When the user submits the form (by clicking a submit button or pressing Enter), the data entered into the form elements is packaged into an HTTP request and sent to the URL specified in the action attribute of the <form> tag. This is the internal mechanism we were talking about above.
